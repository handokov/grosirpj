# Worklog - Task 3: Add Deterministic IDs to seedDemoData()

## Summary
Fixed the issue where `seedDemoData()` in `/home/z/my-project/src/lib/db.ts` created database records with random IDs generated by Prisma's `@default(cuid())`. On Vercel's ephemeral SQLite, different function instances would generate different IDs, causing foreign key mismatches (e.g., product IDs in user carts not matching across instances).

## Root Cause
Prisma's `@default(cuid())` generates a unique random ID on each insertion. On Vercel, each serverless function instance may seed the database independently, leading to different IDs being generated. This breaks referential integrity when one instance creates a product with ID `abc123` and another instance references product ID `def456`.

## Fix Applied
Added deterministic (predictable) IDs to ALL create calls in the `seedDemoData()` function, ensuring that every serverless instance generates the same IDs.
